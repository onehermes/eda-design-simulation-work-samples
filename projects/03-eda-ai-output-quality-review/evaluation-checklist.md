# Evaluation Checklist

- [ ] Does the answer include a ground reference?
- [ ] Are units explicit and unambiguous?
- [ ] Is the simulator named?
- [ ] Are component models or model sources mentioned when needed?
- [ ] Is the analysis type stated clearly?
- [ ] Are ERC and DRC checks included when PCB work is involved?
- [ ] Are validation steps described?
- [ ] Are assumptions and limitations stated?
- [ ] Does the answer avoid claiming that simulation equals a finished PCB?
- [ ] Could another person reproduce the workflow from the instructions?

## Review Standard

An answer should fail review if it leaves out the ground reference, hides key assumptions, or overstates what simulation can prove.
